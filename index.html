Robot should be tested on a web server (e.g Live Server with VSC) to function fully.
Press the On/Off button to turn on the robot.

Actions

It is suggested to wait a couple seconds between actions for animations to complete.

  1. Press anywhere other than the arms and arrows to shoot a laser
  2. Click on either the right arrow or left arrow to make the robot move to that side of the screen
  3. Click on either arm of the robot to make it spin
Once 3/3 actions are performed, the robot will power off. Actions can only be performed while the robot is on and it will only turn fully back on again once the Charge button has been pressed. Turning the robot on and off will maintain the amount of actions left to perform.

You can recharge the robot after action 1, to which the battery will be reset to perform 3 more actions.

My program is optimised to be scaled based on the window size, so the canvas and robot would be appropriately resized based on the device's size. This would make it appropriate and visible enough to use on most if not any device screen. However, if the window size is altered during testing, the web server would need to be refreshed for actions to function as intended.
